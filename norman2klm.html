Clean Comparison (Class & Exam Reference)

Model-based evaluation: what each model explains vs. predicts
Model Category What it Explains What it Predicts
Norman Cognitive / descriptive Understanding, errors, feedback, gulfs ❌ Nothing (no performance prediction)
GOMS Predictive / analytic Task structure for expert users ✔ Relative efficiency
KLM Predictive / quantitative Low-level physical and motor actions ✔ Execution time

KLM Operators (Execution-Level Primitives)

The Keystroke-Level Model (KLM) decomposes the execution phase of interaction into primitive operators. Assumes user is expert and method is already chosen.

KLM operators (Dix et al., Chapter 12)
Operator Name Meaning Example
KKeystrokingPress keyTyping character
BButton pressPress mouse buttonLeft-click
PPointingMove pointer to targetMove cursor to icon
HHomingSwitch hands between devicesKeyboard → mouse
DDrawingDrawing lines using pointing deviceDrag to draw
MMental preparationPrepare for actionPause before click
RSystem responseWait for feedbackApp loading

Worked Example: Correcting a Single Character

Example task: user notices a single-character error, deletes it, retypes it, and returns to insertion point.

  1. Move hand to mouse
  2. Position mouse after wrong character
  3. Return to keyboard
  4. Delete character
  5. Type correction
  6. Reposition insertion point

One possible KLM operator sequence:

H[mouse]
P B[LEFT]
H[keyboard]
M K[DELETE]
K[char]
H[mouse] M P B[LEFT]

This sequence shows how KLM models only execution-level actions. High-level planning, learning, and interpretation are outside its scope.

Formal GOMS Representation

Goal: Enable Wi-Fi

Method: Use Quick Toggle

Operators: Swipe down → Locate Wi-Fi → Tap icon

Selection Rule: If device is unlocked, use Quick Toggle method

Key Takeaways

KLM has seven primitive operators (K, B, P, H, D, M, R) used to predict execution time for expert, error-free interaction.

Norman explains cognitive processes like goals, planning, execution, and evaluation, while GOMS and KLM focus on performance prediction.

Use color, strong, and em to highlight important cognitive points, goals, methods, and exceptions in your reports.
