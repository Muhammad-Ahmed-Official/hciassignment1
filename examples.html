Expense Input

Scenario: You paid $60 and want to split among 3 friends.







Result: Each friend owes $20

Norman – Cognitive Model (7 Stages)

Applying Norman's 7-stage action model to the expense-splitting task:

  1. Forming the Goal: You want to split $60 among 3 friends fairly.
  2. Forming the Intention: Decide to calculate each friend’s share.
  3. Specifying the Action: Enter the total amount and number of friends into the form.
  4. Executing the Action: Type values into the input fields and click the "Calculate Share" button.
  5. Perceiving the State of the World: Observe the result displayed below the form.
  6. Interpreting the Perception: Understand that each friend owes $20.
  7. Evaluating the Outcome: Check if the result matches the intended goal (correct split).

This detailed mapping shows how Norman’s cognitive principles apply to a simple UI task.

GOMS – Goal, Operators, Methods, Selection Rules

Breaking down the expense-splitting task for an expert user:

  1. Goal: Calculate how much each friend owes.
  2. Method: Use the web form: input total amount, input number of friends, click "Calculate".
  3. Operators:
    • Move pointer to "Total Amount" field
    • Type total amount ($60)
    • Move pointer to "Number of Friends" field
    • Type number of friends (3)
    • Move pointer to "Calculate" button
    • Click "Calculate"
    • Observe result ($20 per friend)
  4. Selection Rules:
    • If any field is empty, prompt the user to fill it (error handling not implemented)
    • Use keyboard shortcuts only if mouse is unavailable

This shows the structured steps GOMS focuses on for expert, predictable task execution.

KLM – Keystroke-Level Model

Low-level execution steps for expert user:

[M] Decide total amount
[P] Point to "Total Amount" field
[K] Type total ($60)
[P] Point to "Number of Friends" field
[K] Type number of friends (3)
[P] Point to "Calculate" button
[B] Click button
[R] Observe each friend's share ($20)
