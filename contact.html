How This Page Follows Norman’s Model and GOMS

Norman’s Seven Stages of Action

This contact form is designed according to Norman’s Action Cycle. The goal of the user is to send a message about HCI concepts.

  • Goal Formation: User wants to contact about HCI topics.
  • Intention: User decides to fill out the contact form.
  • Action Specification: Clear labels guide what to type in each field.
  • Execution: User enters name, email, selects topic, writes message, and presses “Send Message”.
  • Perception: Required fields and structured layout help users see what is needed.
  • Interpretation: Button color (green) signals positive action.
  • Evaluation: Form structure reduces confusion, narrowing the Gulf of Execution and Evaluation.

GOMS Analysis

The task of sending a message can be analyzed using GOMS (Goals, Operators, Methods, Selection Rules):

  • Goal: Send a message about an HCI concept.
  • Operators: Typing, clicking, selecting from dropdown, pressing submit.
  • Method: Fill required fields → Select topic → Choose background → Write message → Click “Send”.
  • Selection Rule: If topic is known, select it directly; otherwise choose “Other HCI Concept”.

The structured layout minimizes cognitive load and supports efficient task completion, making it suitable for KLM time prediction and usability evaluation.
